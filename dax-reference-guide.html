📅 Date Table (Calculated Table)
Create as a New Table in Power BI. Mark as Date Table after creating (Table Tools → Mark as Date Table → select [Date]).
Calculated TableRequired First
Date_Table =
ADDCOLUMNS(
    CALENDAR( DATE(2024, 1, 1), DATE(2027, 12, 31) ),
    "Year",           YEAR([Date]),
    "Month Num",      MONTH([Date]),
    "Month Name",     FORMAT([Date], "MMMM"),
    "Month Short",    FORMAT([Date], "MMM"),
    "Quarter",        "Q" & QUARTER([Date]),
    "Week Number",    WEEKNUM([Date]),
    "Day of Week",    WEEKDAY([Date], 2),
    "Day Name",       FORMAT([Date], "dddd"),
    "Is Weekend",     IF(WEEKDAY([Date], 2) >= 6, TRUE, FALSE),
    "Month Year",     FORMAT([Date], "MMM YYYY"),
    "Year Month Num", YEAR([Date]) * 100 + MONTH([Date])
)

Pulls unique last names from both agent tables. Agent Name is already trimmed to last name by Power Query. Use as the "one" side of relationships to both agent tables.

📆 Pay Period# Info (Already in Power Query)
Your Pay Period# is already calculated in Power Query — no DAX calculated column needed. Anchor date is Jan 11, 2026. PP Label is also created there. Shown here for reference.
Agent_Performance_SummaryPower Query Reference
-- These already exist as columns from Power Query — don't recreate in DAX:
--   [PPE Date]     → Pay period end date (date type)
--   [Pay Period#]  → Integer, anchored to 2026-01-11
--   [PP Label]     → "PP1", "PP2", etc.

-- If you ever need to rebuild Pay Period# as a DAX calculated column:
Pay Period# =
    Number.RoundDown(
        Duration.Days([Interval Start] - #date(2026,1,11)) / 14
    ) + 1

-- PP Label DAX version:
PP Label = "PP" & Text.From([Pay Period#])

🔧 Formatting Helpers

Reusable patterns for converting seconds, building dynamic titles, conditional formatting, and other display utilities.

Format Seconds → H:MM:SS
For status durations (Available, Logged In, etc.) Replace the VAR with your actual measure name.
Any TableVAR / FORMAT
Available Time Display =
VAR TotalSec = [Total Available (s)]        -- ← swap in your measure
VAR Hours    = INT( TotalSec / 3600 )
VAR Minutes  = INT( MOD( TotalSec, 3600 ) / 60 )
VAR Seconds  = MOD( TotalSec, 60 )
RETURN
    FORMAT(Hours,   "0")    & ":"
  & FORMAT(Minutes, "00")   & ":"
  & FORMAT(Seconds, "00")
Format Seconds → M:SS
For call metrics like Avg Handle Time, Avg Talk Time where values are under an hour.
Any TableVAR / FORMAT
Avg Handle Time Display =
VAR TotalSec = [Avg Handle Time (s)]        -- ← swap in your measure
VAR Minutes  = INT( TotalSec / 60 )
VAR Seconds  = MOD( TotalSec, 60 )
RETURN
    FORMAT(Minutes, "0") & ":" & FORMAT(Seconds, "00")

Safe DIVIDE Template
Always use DIVIDE instead of "/" to prevent division-by-zero errors. Third argument controls what returns when denominator is zero.
Any TableDIVIDE
-- Returns 0 when denominator is zero (good for rate cards)
My Rate % =
DIVIDE( [Numerator Measure], [Denominator Measure], 0 )

-- Returns BLANK when denominator is zero (good for charts — hides bar)
My Rate % =
DIVIDE( [Numerator Measure], [Denominator Measure], BLANK() )
Color Status Measure (Conditional Formatting)
Returns a hex color based on whether a metric meets a target. Apply via Format → Conditional Formatting → Field value.
Any TableSWITCH / IF
-- Example: SL% color — adjust thresholds to match your targets
SL Color =
VAR SL = [Service Level %]
RETURN
SWITCH(
    TRUE(),
    SL >= 0.80,  "#2ecc87",   -- green: at or above 80%
    SL >= 0.65,  "#f39c12",   -- amber: 65-79%
    "#C4272E"                  -- red: below 65% (department red)
)
